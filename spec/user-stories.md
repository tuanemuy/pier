# ユーザーストーリー — Pier（モバイルネイティブ tmux/SSH クライアント）

最終更新: 2026-07-11
表記: ⭐ = MVP対象。番号は恒久ID（参照用に変更しない）。

MVP対象: A1–A3, A5, B1–B6, C1–C4, D1–D2, E1–E4, F1–F2, G1–G2, I1–I2（計23）

## A. 接続とホスト管理

- **A1⭐** ホスト（アドレス・ユーザー・鍵）を登録し、一覧から1タップで接続できる
- **A2⭐** SSH鍵をアプリ内で生成し、公開鍵をコピーしてサーバーに登録できる（秘密鍵はSecure Enclave/Keychainから出ない）
- **A3⭐** 接続時に既存のtmuxセッション一覧が表示され、選んでアタッチできる。セッションがなければ新規作成される（内部的には `tmux -CC` 経由）
- **A4** パスワード認証・ポート・ProxyJump等の詳細設定ができる
- **A5⭐** 既存セッションにアタッチしても、ウィンドウ・ペイン構成とスクロールバックが正しく復元される（デスクトップで作ったセッションも含む）
  - 実装ノート: tmux -CC はアタッチ時に画面内容を流さないため、`capture-pane -e -p` による復元が必要。F1と同一基盤
- **A6** 接続時のデフォルト動作を選べる（毎回一覧 / 特定セッションに直行 / 常に新規）

## B. セッション / ウィンドウ / ペイン

- **B1⭐** サイドバーでセッション→ウィンドウのツリーを見て、タップで移動できる
- **B2⭐** ウィンドウをタブとして切り替え・新規作成・削除できる
- **B3⭐** ペインはカードデッキとして表示され、上下左右のスワイプで隣のペインへ移動できる
- **B4⭐** ペインのない方向へスワイプすると、その方向に新規分割できる（`split-window -h/-v/-b`）
- **B5⭐** 覗いているカードの縁で、となりのペインの存在がわかる
- **B6⭐** ペインを閉じる・別ウィンドウへ移動できる
  - 実装ノート: カード長押しのコンテキストメニュー。「閉じる」= `kill-pane`、「移動」= `break-pane` / `join-pane`
  - 受け入れ条件: 実行中プロセスがあるペインを閉じる際は確認ダイアログを出す
- **B7** デスクトップから同じセッションに同時接続しても互いに壊れない

## C. コマンド実行（チャットUI）

- **C1⭐** シェルのペインでは、コマンドと出力が1ブロック（吹き出し）として表示される（OSC 133 シェル統合）
- **C2⭐** ブロックには終了コード・実行時間が表示され、成功/失敗が色でわかる
- **C3⭐** 長い出力は折りたたまれ、タップで展開できる
- **C4⭐** 過去のブロックから再実行・コマンドコピー・出力コピーができる
- **C5** 実行中のコマンドは進行中とわかり、Ctrl+Cで中断できる
- **C6** コマンド履歴を検索して呼び出せる

## D. ファイル編集

- **D1⭐** `vim file.txt` のようなファイルを開く操作、またはファイルパスのタップで、ネイティブエディタが開く
- **D2⭐** 開いたファイルを快適に編集し、保存するとリモートに書き戻される
  - 実装ノート: MVPはプレーンテキスト編集に特化（システムUITextViewベース）。等幅フォント・日本語IME・undo/redo・キーボードアクセサリ連携の信頼性を最優先。シンタックスハイライトはD5（MVP外）
  - 受け入れ条件: 日本語の変換・確定・undoが破綻しないこと
- **D3** ファイルブラウザでディレクトリを辿ってファイルを開ける
- **D4** 保存時に権限がない場合、sudoでの保存を提案される
- **D5** シンタックスハイライト付きで編集できる
  - 実装ノート: UITextView + Neon（`TextViewHighlighter` がUITextViewを直接サポート）を第一候補とし、導入時にSTTextView + Neon / Runestoneと再評価する

## E. TUIフォールバック

- **E1⭐** vim / htop / Claude Code などのTUIはターミナルビューでそのまま表示される
- **E2⭐** フォントサイズが画面幅に自動フィットし、ピンチで調整できる
- **E3⭐** Esc / Ctrl / Tab / 矢印などをアクセサリバーから送信できる
- **E4** TUI検出時に入力バーが「キー直接送信モード」に切り替わる（⭐に準ずる。E1〜E3とセットで実装）

## F. 切断耐性

- **F1⭐** アプリを閉じて戻っても、数秒で再接続し完全に元の状態に復帰する（A5と同一基盤）
- **F2⭐** 切断中に実行していたプロセスは失われない（tmuxがサーバー側で保持）
- **F3** 回線切替（WiFi⇔モバイル）でも操作が途切れにくい

## G. 日本語・入力

- **G1⭐** チャット入力バーでは日本語IMEがそのまま使える
- **G2⭐** TUIペインへ日本語を入力できる
  - 実装ノート: クライアント完結方式。TUIモードでも入力バーをコンポーズ用フィールドとして使い、IME確定後の文字列を `send-keys -l`（リテラル）で送信。サーバー側への追加インストールは不要
  - 既知の制約: TUI内でのインライン変換表示は不可（打つ→確定→送信のフロー）。受け入れ条件に明記

## H. 通知（将来）

- **H1** 長時間コマンドの完了やClaude Codeの入力待ちをプッシュ通知で受け取れる

## I. 設定

- **I1⭐** カラーテーマを選べる（ダーク数種・ライト。ターミナル256色/true colorパレットも連動）
- **I2⭐** ターミナル/エディタのフォントと基本サイズを選べる（等幅フォント同梱、日本語グリフのフォールバック含む）
  - 受け入れ条件: 全角（2幅）グリフの描画検証を含む
- **I3** チャットブロックの表示密度や折りたたみ閾値を調整できる
- **I4** アクセサリバーのキー配置をカスタマイズできる
- **I5** ホストごとに設定を上書きできる（本番サーバーだけ赤系テーマ等）
- **I6** 設定をiCloudで同期できる

## スコープ外（意図的に含めていないもの）
